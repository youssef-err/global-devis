## Supabase (auth + cloud invoices)

1. Create a project on [Supabase](https://supabase.com) and copy **Project URL** + **anon public** key.
2. Add them to `.env.local` (see `.env.example`).
3. In **SQL Editor**, run `supabase/migrations/001_invoices.sql` to create the `invoices` table and RLS policies.
4. Under **Authentication → URL configuration**, set:
   - **Site URL**: `http://localhost:3000` (or your production URL)
   - **Redirect URLs**: include `http://localhost:3000/auth/callback`

Invoices stay in **localStorage** as offline fallback; when signed in, changes **debounce-sync** (500ms) to Supabase.
